## Regla de Oro de Implementación 🌟

**NUNCA comiences una implementación sin antes leer y entender el contexto.**

Antes de escribir una sola línea de código o realizar cualquier cambio en la arquitectura, es **OBLIGATORIO** revisar las siguientes fuentes de verdad en el workspace:

1.  **`.agents/`**: Para entender las capacidades y roles de los agentes con los que colaboras.
2.  **`.rules/`**: Para conocer las reglas de arquitectura, estilo y estándares de desarrollo del proyecto.
3.  **`skills/`**: Para verificar si existen habilidades especializadas que debas usar para tareas específicas.

Si no conoces las reglas, no puedes construir correctamente. **Primero lee, luego construye.**
