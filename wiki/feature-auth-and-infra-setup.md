# Feature Spec: Authentication Infrastructure and Core Setup

## 1. Overview
The goal of this feature is to establish the foundational authentication and database infrastructure for the Shopify Sync SaaS project. This involves setting up a robust backend with NestJS and TypeORM, and a seamless authentication experience using NextAuth.js (v5) that works across both the frontend and backend.

## 2. Backend Requirements (NestJS)

### 2.1 Database Setup
- **Technology:** PostgreSQL.
- **ORM:** TypeORM.
- **Task:** 
    - Configure TypeORM to connect to a PostgreSQL instance.
    - Set up the initial database connection configuration (environment variables for host, port, username, password, database name).
    - Ensure the backend can perform basic CRUD operations via TypeORM once entities are defined.

### 2.2 Authentication Integration
- **Technology:** NextAuth.js (v5).
- **Task:** 
    - Implement a mechanism to validate/share authentication state between the backend and frontend.
    - (Note: This might involve JWT validation or a shared session strategy that the NestJS backend can verify).

## 3. Frontend Requirements (Next.js)

### 3.1 Authentication Integration
- **Technology:** NextAuth.js (v5).
- **Task:** 
    - Install and configure NextAuth.js (v5) in the Next.js application.
    - Set up providers (e.g., Credentials provider or OAuth if planned).

### 3.2 Routing and Access Control
- **Public Routes:** Create at least one public landing page (e.g., `/`).
- **Protected Routes:** 
    - Implement a middleware or layout-based protection for any route starting with `/dashboard/*`.
    - Ensure that unauthenticated users are redirected to a login page when attempting to access `/dashboard/*`.
    - Ensure that authenticated users can access the dashboard.

## 4. Technical Details & Constraints
- **Monorepo Structure:** Use the existing Turborepo/pnpm workspace.
- **Shared Config:** Environment variables for database and auth secrets should be managed carefully (using `.env` files in respective apps or a shared workspace config).
- **Authentication Flow:** The authentication must be "connected" so the backend can trust the sessions/tokens generated by NextAuth on the frontend.

## 5. Definition of Done (DoD)
- [ ] Backend successfully connects to PostgreSQL using TypeORM.
- [ ] Backend can verify an authentication session/token.
- [ ] Frontend is configured with NextAuth.js v5.
- [ ] A user can log in and access a page under `/dashboard/`.
- [ ] An unauthenticated user is blocked from `/dashboard/` and redirected.
- [ ] All new dependencies are correctly added to `package.json` in their respective apps.
